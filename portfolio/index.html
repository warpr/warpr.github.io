MusicBrainz

MusicBrainz is a community-maintained open source encyclopedia of music information. As a contractor on MusicBrainz I've mostly worked on the musicbrainz.org server software, which is written in perl using the Catalyst framework. Some of the data entry screens use javascript with jQuery and Knockout. The main site uses PostgreSQL as the back-end database and a lucene based search server written in java.
date February 2010 — July 2013
url https://musicbrainz.org/
tech perl javascript postgresql

LicenseDB

The License Database is intended to be a metadata database about open source / free software / free culture copyright licenses. The database is implemented as a set of git versioned turtle files, from which HTML/RDFa, RDF/XML and JSON-LD files are generated. A hodge podge of scripts glue everything together. I would like to reimplement all of this in a more coherent manner.
date February 2012 —
url https://licensedb.org/
tech RDF node.js Javascript Python PHP

databank-couchdb

databank is Evan Prodromou's NoSQL abstraction layer for node.js, which is used by the pump.io social network server.

I have written a databank driver for CouchDB. It is feature complete but not ready for production use. So far the work on this has been entirely straightforward, the interesting part will be dynamically generating permanent views (CouchDB's equivalent to indexes). It currently uses temporary views, which are inherently slow.

date January 2013, May 2013
url https://github.com/warpr/databank-couchdb/
tech node.js Javascript CouchDB

DATA×DATA

Most web analytics systems are designed to track users, which raises privacy concerns. DATA×DATA is designed to only log screen resolution and browser version for each page view, which is what I'm interested in as a web developer.
date January 2013
url https://frob.nl/DATAxDATA/
tech node.js Javascript MySQL

senet.org

senet.org is intended to be a metadata database about videogames. This project was largely about experimenting with RDF, so storing data in the database in way that makes it accessible for webservice clients as e.g. RDF/XML or JSON-LD or for website visitors as RDFa. But more importantly trying to express with RDF how a data entry form should be rendered and validated.
date April 2012 — October 2012 (unfinished)
url https://gitorious.org/senet/
tech RDF Javascript Python CouchDB

autoincrement.org

autoincrement.org is a silly domain originally intended as a way to mint user-friendly identifiers for documents stored in NoSQL databases which do not have an autoincrement type or sequence common in SQL databases. I now just re-implement it whenever I want to learn a new web framework or programming language. Past implementations have been in Python and Opa, the current version is written in Haskell.
date November 2012
url http://autoincrement.org/
tech Haskell MySQL

Testloop

testloop was intended to GUI testrunner based around the idea of continuous testing. The idea is to use per-test coverage data to predict which tests will likely fail after a particular change to the source tree, and then instantly run those tests instead of simply running the entire test suite continuously. I got this to a proof-of-concept stage but then abandoned it, as continuous integration tools like jenkins are often Good Enough.
date August 2011 — April 2012 (unfinished)
url https://testloop.org/
tech Python Qt
